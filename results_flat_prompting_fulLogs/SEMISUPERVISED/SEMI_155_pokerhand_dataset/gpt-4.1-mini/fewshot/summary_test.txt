 AutoML Process Summarization

### Data Profiling Phase
- Columns: 11
- All columns processed are identified as Integer type.
- No missing values detected.
- No non-numeric features or useless columns found.
- Original dataset size: 663,361 samples, sampled down to 2,000 samples for efficiency.

### Pipeline Search and Synthesis Phase

#### Setup
- Manual grammar created for a semi-supervised learning task.
- Grammar productions defined for pipeline components including Imputer, Encoders, Feature Scaler, Semi-supervised classifiers, and Classifiers.
- Semi-supervised classifier options include:
  - AutonBox
  - SelfTrainingClassifier
  - LabelPropagation variants (SkLabelPropagation and SkLabelSpreading)
- Feature scalers considered include MaxAbsScaler, RobustScaler, StandardScaler or none.

#### Iteration 1 - Pipeline Search with MCTS (Monte Carlo Tree Search)

##### Pipeline Construction and Exploration
- Initial pipeline starts at the start symbol `S`.
- Multiple simulations explore expanding the pipeline by selecting components:
  - Imputer and Encoders mostly set to no-operation (E).
  - Feature Scaler choices tested: MaxAbsScaler, StandardScaler, RobustScaler, and the option of no scaler.
  - Various Semi-supervised classifiers and LabelPropagation classifiers inserted as the core model.
- The coach guides filling pipeline slots progressively.
- Retracts unnecessary components (like Encoders and Imputer) when found non-beneficial.
  
##### Pipeline Evaluation Attempts
- Pipelines are constructed and scored via cross-validation.
- During scoring, some pipelines cause exceptions:
  - A pipeline with a custom semi-supervised classifier attempts to transform data via `.toarray()`, but the input is a pandas DataFrame, causing an AttributeError.
  - Others fail due to memory issues when trying to compute kernel matrices in LabelPropagation:
    - Specifically, an attempt to allocate over 2 TiB memory for a kernel matrix on large data leads to an `ArrayMemoryError`.

##### Result of the Pipeline Search
- The search explores many pipeline variations including feature scaling and label propagation classifiers.
- Despite pipeline creation and attempt of cross-validation scoring, all pipelines fail to be robustly scored or selected.
- After timeout and signal to terminate the search, no successful pipelines are recorded.

### Overall Summary

- The AutoML system successfully profiles and processes the dataset, selecting a semi-supervised learning setting.
- It dynamically generates pipelines exploring various preprocessing and semi-supervised classification methods.
- Fundamental issues arise from pipeline scoring:
  - Semi-supervised classifier code incompatibility with DataFrame inputs.
  - Computational resource limitations with large kernel-based methods (LabelPropagation).
- These issues lead to exceptions and ultimately no pipeline is retained or reported as found.
- The process ends with warnings about no pipelines found, indicating the need for pipeline or code refinements to handle the data and resource constraints effectively.

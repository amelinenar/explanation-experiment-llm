 AutoML Process Summarization

### Data Profiling Phase
- Columns: 21
- Numeric features:
  - Integer columns: loc, v(g), ev(g), iv(g), n, lOCode, lOComment, lOBlank, locCodeAndComment, uniq_Op, uniq_Opnd, total_Op, total_Opnd, branchCount (14 columns)
  - Float columns: v, l, d, i, e, b, t (7 columns)
- Missing values detected in dataset.
- Due to dataset size (7484 samples), data was sampled down to 2000 for efficient processing.

### Grammar and Task Setup
- Manual grammar created for SEMISUPERVISED_TASK.
- Grammar includes pipelines with components: IMPUTER, ENCODERS, FEATURE_SCALER, SEMISUPERVISED_CLASSIFIER, CLASSIFIER, and LABELPROPAGATION_CLASSIFIER.
- Available primitives for pipeline components:
  - Imputer: sklearn.impute.SimpleImputer
  - Feature Scalers: MaxAbsScaler, RobustScaler, StandardScaler, or no scaler (E)
  - Semi-supervised classifiers: AutonBox, SelfTrainingClassifier, SkLabelPropagation, SkLabelSpreading
  - Classifiers: various sklearn classifiers and boosted tree classifiers (RandomForest, XGB, LightGBM, etc.)
  - Encoders can be either no-op (E) or other encoding steps.

### Pipeline Search and Synthesis Phase

#### Iteration 1 - Multiple Monte Carlo Tree Search (MCTS) Simulations

- Initial simulations work from the start symbol "S" and explore splits of the pipeline into components.
- Various pipeline structures considered, including:
  - IMPUTER → ENCODERS → FEATURE_SCALER → SEMISUPERVISED_CLASSIFIER → CLASSIFIER
  - IMPUTER → ENCODERS → FEATURE_SCALER → LABELPROPAGATION_CLASSIFIER
- Encoders often replaced with no encoding (E) early in the process.
- Feature scalers tested include no scaler (E), MaxAbsScaler, RobustScaler, and StandardScaler.
- Imputer always replaced from an abstract action to `sklearn.impute.SimpleImputer` with default strategy.
- Semi-supervised classifiers explored:
  - SkLabelSpreading
  - SkLabelPropagation
  - AutonBox (a specialized semi-supervised method using base estimator)
  - SelfTrainingClassifier
- Classifiers Tried:
  - RandomForestClassifier
  - MultinomialNB
  - KNeighborsClassifier
  - LinearDiscriminantAnalysis
  - Others including Gradient Boosting and Naive Bayes.

- Multiple pipelines were built combining these components, and each created pipeline was scored:
  - Early pipelines (with SkLabelSpreading, SimpleImputer, StandardScaler) scored around 0.22
  - Some pipelines with RobustScaler and SkLabelSpreading had scores around 0.21
  - Later pipelines involving combination of SimpleImputer, RobustScaler, AutonBox with RandomForestClassifier scored higher, approx 0.768.
  - Another pipeline with SimpleImputer and SkLabelSpreading also reached a high score of ~0.7588.
  
- Feature scalers were toggled on/off to test the impact (scalers replaced with E in some runs).
  
- Final selected models involved semi-supervised classifiers (AutonBox, SkLabelSpreading) combined with SimpleImputer and various scalers or no scaler and different supervised classifiers.

### Search Termination and Results
- The search process terminated after the timeout was reached (~1 minute and 57 seconds of search).
- At termination, 2 strong candidate pipelines were found and scored successfully with approximately 0.768 and 0.758 metrics.
- The best pipeline involved:
  - Imputation (SimpleImputer with most frequent strategy)
  - Feature scaling (RobustScaler or StandardScaler)
  - Semi-supervised classifier (AutonBox or SkLabelSpreading)
  - Supervised classifier (RandomForest or MultinomialNB)

### Summary

This AutoML run processed a dataset with 21 columns primarily numeric with missing values and applied sampling due to dataset size. It constructed a search grammar tailored for semi-supervised tasks, including a rich set of preprocessing and classification primitives.

The pipeline optimization used Monte Carlo Tree Search to explore pipeline compositions involving imputers, optional encoders, feature scalers, semi-supervised classifiers, and supervised classifiers. Through iterative simulations, it tested multiple component combinations, assessed their predictive performance via scoring, and refined pipeline structure and components accordingly.

The best pipelines combined SimpleImputer for missing values, robust feature scaling (RobustScaler, StandardScaler), advanced semi-supervised classifiers (AutonBox, SkLabelSpreading), and strong supervised classifiers (RandomForest, MultinomialNB). The process concluded by successfully identifying two promising pipelines before the search timeout.

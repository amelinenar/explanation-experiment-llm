 AutoML Process Summarization

### Data Profiling Phase
- Columns: 12
- All columns are identified as integer or float types.
- Several columns are marked as identifiers (e.g., 'd3mIndex', 'ad_id', 'advertiser_id', 'query_id', 'keyword_id', 'title_id', 'description_id', 'user_id').
- No missing values detected.
- Dataset size initially 119,685 rows, downsampled to 2,000 for quicker processing.

### Grammar and Search Setup Phase
- Manual grammar for a **semi-supervised learning task (SEMISUPERVISED_TASK)** was created.
- Grammar includes pipeline components: Imputer, Encoders, Feature Scalers, Semi-supervised Classifiers, and Regular Classifiers.
- Semi-supervised classifiers include specialized built-in primitives (AutonBox) and sklearn models (SelfTrainingClassifier).
- Label propagation classifiers also included, such as `SkLabelPropagation` and `SkLabelSpreading`.

### Pipeline Search (MCTS) and Simulation Phase

#### Iteration 1 - Multiple Simulations
- Initial pipelines begin with symbolic "S" expanded into pipeline components.
- Search explores pipelines with different compositions, e.g., using semi-supervised classifiers or label propagation methods.
- Encoders and imputers frequently pruned as unnecessary ("E" denotes empty or none).
- Feature scalers tested include `MaxAbsScaler`, `StandardScaler`, and `RobustScaler`.
- Label propagation classifiers instantiated as `SkLabelPropagation` and `SkLabelSpreading`.
- Several pipelines created with combinations of scalers and classifiers.

#### Pipeline Instantiation and Scoring
- Creation of new pipelines logged, such as pipelines consisting of:
  - A scaler (e.g., `RobustScaler` or `StandardScaler`).
  - A semi-supervised label propagation classifier (`SkLabelSpreading`).
- Pipeline scoring attempted via cross-validation.

### Errors and Issues Encountered
- **AttributeError** during scoring of a pipeline with `SkLabelSpreading` related to calling `.toarray()` on a pandas DataFrame, indicating incompatible data format or preprocessing needed.
- **Memory error (numpy ArrayMemoryError)** when running RBF kernel computation in label propagation, trying to allocate ~68.3 GiB array due to large dataset dimension (95,748 × 95,748).
- These errors prevented successful scoring of candidate pipelines.

### Search Termination and Outcome
- Search timeout reached after roughly 68 seconds.
- Final output: **No pipelines found** due to scoring and memory errors during pipeline evaluation.

---

### Summary and Insights

1. **Data Understanding:** The dataset consists solely of numeric columns with some identifiers and no missing values, permitting straightforward data input but implying potential dimensionality challenges.

2. **Task Definition:** Semi-supervised learning task is targeted, given the presence of labeled and unlabeled data.

3. **Pipeline Search:** AutoML explores pipelines combining preprocessing steps (optional imputers, feature scalers) and advanced semi-supervised classifiers, focusing on label propagation techniques.

4. **Challenges:**
   - The built semi-supervised classifier pipelines encountered incompatibilities with data types (DataFrame vs expected sparse/dense array).
   - Large dataset size led to out-of-memory errors in affinity matrix computation for label propagation methods.

5. **Outcome:** No valid pipelines successfully scored, resulting in the AutoML process ending without a viable model recommendation.

---

### Recommendations
- Address data type transformations before feeding into semi-supervised classifiers (e.g., convert DataFrame to numpy array or sparse matrix).
- Consider smaller subsets or dimensionality reduction to prevent memory blowup in graph-based semi-supervised methods.
- Possibly disable or limit the use of label propagation or graph-based classifiers for large datasets if memory constraints exist.
